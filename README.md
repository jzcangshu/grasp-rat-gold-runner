# Grasp Rat Gold Runner

给 Grasp Rat Game 写的篡改猴 userscript（用户脚本）。

它不是一个“只会找最近金币”的小脚本，而是一套半自动驾驶：平时替你吃金币，危险时保命，临时交战时帮你躲弹、标记目标，并提供一个尽量不挡视野的游戏式 HUD（抬头显示）。

游戏入口：`https://grasp-rat-game.h-e.top/`

原始玩法说明可参考 Linux.do 帖子：`https://linux.do/t/topic/2290514`。原帖里提到的核心规则很直接：10000m 大地图，20Hz 同步，玩家移速约 10m/s；玩家有 100 HP，命中一次扣 25，死亡会爆出额度，走过去即可吸收。作者也明确说“如果有自动化，那也是欢迎的呢w”。

## 它适合谁

- 想减少重复跑图、专心看局势的人。
- 想研究 Grasp Rat Game 实时状态、坐标、弹体和 UI 自动化的人。
- 想在一个已经正规化的 userscript 项目上继续迭代的人。

如果你只是第一次打开这个项目，记住三件事就够了：

1. `dist/grasp-rat-gold-runner.user.js` 是给篡改猴安装的版本。
2. `src/grasp-rat-gold-runner.user.js` 是开发时改的源文件。
3. 这个脚本已经不使用传送保命；脱战只点击游戏里的“离开”。

## 核心玩法

### 自动巡航

脚本会持续扫描金币，不是简单冲向最近的一个，而是规划一小段“顺路吃多个金币”的路线。

规划时会同时考虑：

- 金币金额和距离。
- 附近有没有金币团。
- 连续吃多个金币是不是会少走回头路。
- 路线附近有没有高 Drop 敌人。
- 当前目标是否还值得继续追。

直观一点说，它更像外卖员规划一段顺路路线，而不是看到最近订单就掉头。

### 危险规避

游戏里黄色 `Drop` 数代表敌人死亡后可能掉落的金币。脚本把它当作风险信号，而不是看账户金币之类的字段。

- 视野约 500m：用于识别敌人和画蓝色敌人线。
- 红圈射程约 150m。
- 170m 是脚本的危险缓冲区。
- Drop 大于 10 的敌人会被当作高价值也高风险目标。
- Drop 不高但最近 10 秒移动过的敌人，进入 170m 也会触发紧急远离。

常态巡航下，如果你真正受到伤害，脚本会立即点击“离开”脱战。左侧官方栏出现 `1h体力限制` 时，也会自动离开。

### 临时交战

左下角可以打开“临时交战”模式。开启后，金币巡航会暂停，脚本进入战斗辅助状态：

- 识别游戏弹体，按弹体起点、方向、速度和游戏 tick（游戏刻）预测轨迹。
- 选择尽量远离弹道的移动方向，但不会和你的 WASD 手动输入抢控制权。
- 趁弹体压力较低时，把你和近身敌人的距离拉回 100-150m。
- 给 170m 内血量低于你的敌人头上画红色倒三角。
- 血量较低时加重红色呼吸灯提醒。
- HP 不超过 9 时立即点击“离开”脱战。

交战模式允许受伤，所以它不会复用常态巡航的“HP 一掉就离开”规则。

### 自动攻击

左上角 `ATTACK BUFFER` 上方有独立的“自动攻击”开关，默认关闭。

开启后，脚本会在红圈射程内自动选择 HP 最低、距离最近的敌人，并根据目标移动轨迹、距离和估算子弹速度做短时预判。攻击不是单发点射，而是长按连发覆盖：每组压住鼠标发 5-8 发，让远距离目标附近形成更密的弹幕覆盖。

`ATTACK BUFFER` 列表会显示 170m 内敌人的用户名、HP 和距离。点击某个名字可以锁定攻击对象；只要它还在 500m 视野内且存活，脚本不会自动换目标。目标离开视野或死亡后，锁定会自动解除。

点击“停止”或“离开”会自动关闭自动攻击。

### 追杀与手动目标

左下角支持输入用户名片段并开启“自动追杀”。脚本只按用户名匹配，不按 user_id（用户 ID）匹配。匹配后会根据目标最近运动轨迹做短时预判并追赶。

你也可以在游戏画布上右键，设置一个临时前往坐标。右键目标优先于金币巡航，但低于受伤离开、死亡停止、体力检查和必要的安全规避。

## HUD 长什么样

HUD 会避开游戏左侧官方面板，自适应放在右侧游戏场景区，不遮挡中心视野。

主要信息包括：

- 当前模式：待机、巡航、追杀或交战。
- 当前目标：金币路线、右键坐标、追杀对象或交战状态。
- 危险状态：近身敌人、弹体数量、低血提醒等。
- `DROP TOP 5`：全场 Drop 前五名，每 30 秒刷新；点击用户名可复制。
- 连线指示：金色是当前目标，蓝色是 500m 内 Drop 大于 0 的敌人，红色是 170m 内危险敌人。

为了性能，HUD 使用普通半透明背景；连线和标记使用单个透明 canvas（画布）绘制，不做毛玻璃滤镜，也不反复重建大量 DOM。

## 安装

搭配 Tampermonkey（篡改猴）使用：`https://www.tampermonkey.net/`

安装脚本内容使用 `dist/grasp-rat-gold-runner.user.js`。

```text
grasp-rat-gold-runner/
  src/
    grasp-rat-gold-runner.user.js   # 源码，开发改这里
  dist/
    grasp-rat-gold-runner.user.js   # 发布版，安装到篡改猴用这里
  docs/
    behavior.md                     # 详细行为规则和优先级
    install.md                      # 安装说明
    development.md                  # 开发与发布流程
    changelog.md                    # 版本记录
  scripts/
    sync-dist.ps1                   # 同步 src 到 dist
    release-check.ps1               # 发布前检查
  AGENTS.md                         # 给后续 agent 的项目手册
```

## 开发

改功能时优先改 `src/grasp-rat-gold-runner.user.js`。发布前同步到 `dist`：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\sync-dist.ps1
```

语法检查：

```powershell
node --check .\src\grasp-rat-gold-runner.user.js
node --check .\dist\grasp-rat-gold-runner.user.js
```

完整发布检查：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\release-check.ps1
```

## 给下一位维护者

这个项目最容易踩坑的地方不是语法，而是各个自动化层的优先级：

- 生命安全高于移动目标。
- 临时交战高于金币巡航。
- 自动攻击是独立火控层，不应该改变移动分支。
- 连线和倒三角必须使用游戏原生坐标换算，不能按屏幕中心硬猜。
- Drop 判断只能看黄色 Drop 对应字段，不能用账户金币字段。

详细规则请先读 `AGENTS.md` 和 `docs/behavior.md`，再动源码。
