# 手机端脚本说明

手机端脚本是独立 userscript，不把移动端判断塞进 PC 脚本。入口文件：

- `src/grasp-rat-gold-runner-mobile.user.js`
- `dist/grasp-rat-gold-runner-mobile.user.js`

## 适配取舍

手机屏幕小，主要矛盾不是展示更多数据，而是少挡视野、减少误触、保留关键控制。因此 mobile 版本做了这些取舍：

- 删除桌面版正下方、左上角、右上角的只读数据框。
- 顶部只保留四个高频按钮：启动、停止、交战、离开。
- 长按游戏画布设置手动坐标目标，替代 PC 右键选点。
- 有手动目标时，右上角显示“取消目标”按钮。
- 火控锁定列表和追杀列表默认收起，点击后以较扁的浮窗显示在顶部按钮下方。
- 追杀列表只展示全场 Drop 前五名，显示用户名、Drop 数和距离；点击用户名只负责填入追杀输入框，是否开启追杀由用户再点“追杀”确认。
- 保留普通半透明控件背景，但取消桌面版蓝色全屏 HUD 风格层。
- 字号和控件间距按手机屏幕缩小，避免遮挡中心视野。

## 逻辑边界

mobile 版本复用核心状态读取和安全优先级：受伤离开、1h 体力限制离开、170m 危险缓冲、金币路线规划、临时交战、自动攻击和追杀仍按原优先级运行。

UI 和输入层是独立的：不要把 mobile 的长按选点、顶部浮窗或紧凑布局加回 PC 脚本。
