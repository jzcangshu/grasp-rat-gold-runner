# Changelog

## 1.6.9

- 新增左侧栏 `1h体力限制` 自动离开：每秒读取游戏官方 `.side` 面板文本，命中后自动点击“离开”退出游戏。
- 该检测独立于金币巡航主循环，并且只读取左侧栏，不读取聊天区，避免历史聊天误触发。

## 1.6.8

- 新增“自动追杀”模式：输入玩家 ID 片段后模糊匹配玩家 `user_id`，自动追向目标。
- 追杀目标优先使用实时可见实体；不可见时使用 minimap 快照里的最近已知点。
- 追杀移动加入短时预判：优先使用实体 `vx/vy`，否则用最近观察位置差估算速度。
- 明确追杀优先级：低于全局生命/体力安全和临时交战，高于普通富敌规避、右键坐标和金币巡航。

## 1.6.7

- 移除 HUD 所有 `backdrop-filter` / `-webkit-backdrop-filter` 毛玻璃效果，改为普通半透明背景，降低渲染开销。
- 新增 `.gitattributes`、开发流程文档和发布检查脚本，规范版本、发布与 git 提交流程。
- `package.json` 增加 `sync:dist` 和 `release:check` 脚本。

## 1.6.6

- 手动开启临时交战模式时，自动取消开启前已有的右键坐标目标。
- 临时交战已开启后，用户重新右键设置的新目标仍然保留，并继续优先于自动躲避/远离危险敌人。

## 1.6.5

- 临时交战模式不再触发常态全局受击自动传送；交战中允许受伤，仍保留 `HP≤25` 加强红色呼吸灯和 `HP≤9` 自动脱战传送。
- 传送体力不足检查增加来源标记：交战模式下只有低血量脱战传送失败才自动点击“离开”，其他来源只记录体力不足状态。
- 更新行为文档和 agent 约束，明确常态巡航与临时交战的血量安全优先级差异。

## 1.6.4

- 修复 1.6.3 后连线器和临时交战红色倒三角仍偏移的问题。
- 坐标换算改为优先复用游戏原生 `worldToScreen` / `viewParams`，再转换到 HUD 右侧透明 canvas 内部坐标；兜底路径也改为按游戏相机中心、左侧保留区和当前视野半径计算。
- 修复临时交战弹体识别：游戏弹体没有实时 `x/y` 字段，脚本现在会按 `start_x/start_y`、方向、速度和 tick 还原当前位置与弹道。
- 弹体来源新增原生 `getRenderBullets()`，并对同一 `bullet_id` 去重，避免同一颗弹体被重复计入躲避评分。

## 1.6.3

- 修复连线器和临时交战倒三角的系统性偏移。
- overlay 坐标 fallback 现在以 HUD 右侧游戏场景区为基准，不再使用全屏 canvas 中心。

## 1.6.2

- 临时交战模式中，右键坐标目标现在优先于自动躲避/远离危险敌人。
- 抽出右键目标移动逻辑，巡航和临时交战共用同一套到达/移动/清除目标处理。
- 修复鼠标/准星可能闪到屏幕固定位置的问题：连线渲染不再调用 `setPointerFromClient`，该函数只保留给真实右键点击取坐标使用。

## 1.6.1

- 修复临时交战模式与手动 WASD/方向键输入抢控制的问题。
- 拆分脚本移动键和用户移动键；交战模式检测到手动移动时暂停自动躲避下发移动，只保留观察和标记。
- HUD 在手动接管时显示 `MANUAL` 状态。

## 1.6.0

- 左下角按钮从“自保传送”改为“临时交战”模式开关。
- 临时交战开启后暂停金币巡航和右键坐标前往，禁用所有金/蓝/红连线器。
- 新增弹体扫描与保守躲避评分，优先选择远离多条弹体轨迹且减少频繁换向的移动方向。
- 170m 内低血敌人头上显示红色倒三角优先攻击提示。
- 临时交战中新增血量风险灯：落后近身敌人 15% 以上显示红色呼吸灯，HP 不超过 25 时加强，HP 不超过 9 时立即传送，体力不足则离开。

## 1.5.2

- 连线显示过滤 `Drop` 为 0 的敌人，蓝线和红线只绘制 `Drop` 大于 0 的敌人。

## 1.5.1

- 连线渲染从 100ms 定时刷新 SVG 改为 canvas + requestAnimationFrame 帧同步绘制。
- 避免每次刷新用 `innerHTML` 重建线段 DOM，降低卡顿和布局开销。
- 坐标换算缓存缩短，并根据玩家移动动态修正中心点，减少连线跳动。

## 1.5.0

- 170m 紧急逃离改为针对黄色 `Drop` 大于 10 的敌人，或 `Drop` 不超过 10 但最近 10 秒内移动过的敌人。
- 新增三类实时连线：金色目标线、蓝色 500m 敌人线、红色 170m 危险线。
- 连线层使用透明 SVG 覆盖，不接管鼠标事件，也不增加全屏毛玻璃背景。

## 1.4.1

- 修正富敌识别机制。
- 富敌判断现在只使用黄色 `Drop` 值：`death_reward_preview ?? death_drop_coins ?? 0`。
- 移除 `coins`、`death_total_loss_preview` 对威胁判断的影响，避免把账户/总金币误当成 Drop。

## 1.4.0

- 新增鼠标右键手动坐标目标。
- 右键游戏画布时读取当前世界坐标并自动前往。
- 手动目标遵守原安全优先级，受伤传送和敌人规避仍然优先。
- 到达目标附近后自动清除手动目标，恢复金币规划。
