# Auto Fire Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add an optional auto-fire mode that targets the lowest-HP enemy inside the 150m red circle and fires with motion prediction and stamina-aware rate limiting.

**Architecture:** Keep auto-fire independent from movement decisions. It runs from the existing main loop after survival checks, selects a live enemy inside 150m, predicts intercept direction from target motion and projectile speed, converts world target to client coordinates through the existing adaptive camera conversion, then dispatches mouse events only when the stamina/rate limiter allows. It must not use fixed screen coordinates and must not replace manual movement priority.

**Tech Stack:** Tampermonkey userscript, page-context JavaScript, existing game globals (`state`, `canvas`, `worldToScreen`, `viewParams`, `setPointerFromClient`).

### Task 2: Implement Prediction And Click

**Files:**
- Modify: `src/grasp-rat-gold-runner.user.js`

**Steps:**
1. Select the lowest-HP live enemy inside 150m, breaking ties by distance.
2. Estimate enemy velocity from entity velocity fields or existing enemy motion history.
3. Solve a short intercept lead using projectile speed, clamped to avoid extreme prediction under latency.
4. Convert predicted world target to client coordinates using existing world-to-client helper.
5. Dispatch `mousemove`, `mousedown`, `mouseup`, and `click` events on the game canvas, with `setPointerFromClient` as a firing-time alignment fallback.

### Task 3: Add Stamina-Aware Rate Limiting

**Files:**
- Modify: `src/grasp-rat-gold-runner.user.js`

**Steps:**
1. Treat one shot as 0.5 stamina.
2. Never exceed one shot per 100ms.
3. Scale cooldown upward as 5s stamina approaches empty; do not shoot when estimated stamina is below one shot.
4. Surface the current auto-fire status in the HUD action/status text.
