# 行为逻辑说明

## 坐标与单位

游戏内部坐标以厘米为单位。脚本中的距离常量也按厘米写：

- `25000` = 250m
- `22000` = 220m
- `17000` = 170m

## 威胁判定

脚本把“富敌”定义为黄色 `Drop` 数量大于 10 的敌方存活实体。

`Drop` 对应游戏实体字段：

```js
death_reward_preview ?? death_drop_coins ?? 0
```

不要用 `coins` 字段判断富敌。`coins` 可能表示账户/总金币类数值，不是画面黄色 `Drop` 标签。

170m 紧急逃离使用更宽的“近身威胁”定义：

- 黄色 `Drop` 大于 10。
- 黄色 `Drop` 不超过 10，但脚本在最近 10 秒内观察到该敌人坐标移动过。

低 `Drop` 敌人的移动历史按敌方 `user_id` 记录，只保留最近看到和最近移动时间。

## 规避逻辑

- 富敌在 250m 内：选金币时排除会靠近富敌 220m 内的目标。
- 富敌在 220m 内：主动拉开距离。
- 近身威胁在 170m 内：紧急反向逃离，并显示暗红色呼吸 HUD。
- 没有受到伤害时不点击“离开”脱战。

## 连线指示

- 金黄色轻微发光线：当前将要前往的目标。金币/右键坐标会指向目标点，逃离时会指向反方向的临时逃离点。
- 蓝色轻微发光线：实时视野 500m 内 `Drop` 大于 0 的敌人。
- 红色轻微发光线：170m 危险距离内 `Drop` 大于 0 的敌人。

`Drop` 为 0 的敌人不绘制蓝线或红线，避免画面被低价值实体干扰。

连线层是透明 canvas（画布）覆盖层，使用 requestAnimationFrame（浏览器帧同步）绘制，不带全屏滤镜或实体背景，不会接管鼠标事件。

为避免卡顿，连线层不要用 `innerHTML` 反复重建 SVG DOM。当前实现会在同一个 canvas 上清屏重画，并把设备像素比限制在合理范围内。

连线器和红色倒三角先复用游戏原生 `worldToScreen`（世界到屏幕）/`viewParams`（视图参数）换算到全屏游戏 canvas，再减去 HUD 右侧场景区 root 的左上角作为透明 canvas 内部坐标。不要自己估算全屏中心或 HUD 中心，否则左侧官方面板、缩放和相机中心都会让 overlay 出现系统性偏移。

临时交战模式开启时禁用一切连线器，不绘制金线、蓝线、红线。此时同一个 canvas 只保留 170m 内低血敌人头上的红色倒三角攻击提示。

## HUD 性能策略

HUD 信息块只使用普通半透明 `rgba` 背景。不要使用 `backdrop-filter`（背景滤镜）或 `-webkit-backdrop-filter`（WebKit 背景滤镜），避免毛玻璃造成额外渲染开销。

## Drop 排行榜

HUD 控制区内的 `DROP TOP 5` 常态每 10 秒刷新一次，展示全场黄色 `Drop` 数最高的 5 名玩家。脚本刚进入游戏、首次识别到当前玩家实体后，会在 3 秒后额外刷新一次，避免刚登录时排行榜还停留在空数据。

数据来源会合并两路：

```js
death_reward_preview ?? death_drop_coins ?? 0
state.minimap.points[*].d
```

`state.entities` 可能在不同同步状态下表现为可见实体或较完整实体列表；`state.minimap.points` 的 `u/d/x/y` 会覆盖全场小地图点，其中 `d` 是全场 Drop。排行榜按同一个用户合并两路数据并取更高 Drop。不要用 `coins` 字段做排名。

用户名优先读取实体 `name`，其次读取 `state.userNames`。如果没有真实用户名，排行榜会显示未知用户，但不会把纯 ID 或 `User id` 当成可复制用户名。

点击排行榜里的用户名会把该用户名填入追杀用户名输入框，并同时复制到剪贴板。复制优先使用浏览器 Clipboard API（剪贴板接口），失败时退回到临时 `textarea` 的复制方式；复制失败不影响输入框填入。

## 临时交战模式

左下角“临时交战”按钮用于切换模式。开启后：

- 暂停金币巡航；右键坐标目标仍可使用，并优先于自动躲避/远离危险敌人。
- 手动开启临时交战时，会取消开启前已有的右键坐标目标，避免旧目的地把交战移动带偏。
- 临时交战已开启后，再次右键可以设置新的交战内临时目标。
- 不执行富敌 170m/220m 巡航规避分支，改由交战模式自身处理危险。
- 手动 WASD/方向键输入优先于自动躲避；检测到手动移动时，脚本只清理自己按下的方向键，自动躲避暂停下发移动。
- 扫描游戏原生 `getRenderBullets()` 和常见弹体集合字段，如 `bullets`、`projectiles`、`shots`、`missiles`、`arrows`。
- 对游戏原生弹体，使用 `start_x/start_y`、`dir_x_micros/dir_y_micros`、`speed_per_tick`、`created_tick/expire_tick` 还原当前位置和速度；只有非原生结构才退回最近位置估算。
- 使用多个短时间窗口预测弹体轨迹，选择尽量远离所有弹体轨迹的方向。
- 对换向增加惩罚，减少服务器延迟下频繁左右横跳。
- 自动躲避会同时维护近身敌人的距离带：优先把最近敌人保持在 100-150m，理想距离约 125m。
- 近弹压迫较高时，弹体规避优先；没有弹体或弹体风险较低时，会抓紧向 100-150m 距离带调节，避免越躲越贴脸。
- 170m 内敌人 HP 低于自身 HP 时，在敌人头上画红色倒三角。
- 若 170m 内敌人 HP 高于自身 HP 15% 以上，显示红色呼吸灯。
- 自身 HP 不超过 25 时，使用更大范围、更高频率的红色呼吸灯。
- 自身 HP 低于 22 时，主循环血量检测频次从约 150ms 提高到约 50ms。
- 自身 HP 不超过 9 时，立即点击“离开”脱战。
- 临时交战模式允许受伤，不执行常态模式的“HP 一下降就离开”逻辑。

关闭临时交战模式后，脚本恢复金币巡航和原连线器显示。

## 自动攻击

HUD 左上角 `ATTACK BUFFER` 上方的“自动攻击”按钮用于切换火控层。该开关默认关闭，独立于左下角启动/停止/临时交战按钮；点击“停止”或“离开”会自动关闭。开启后：

- 不改变移动模式优先级；金币巡航、追杀、右键坐标和临时交战仍按各自规则移动。
- 生命/体力安全优先：死亡、非存活、常态受伤离开、交战 `HP<=9` 离开和短时体力耗尽都高于自动攻击。
- HUD 左上侧 `ATTACK BUFFER` 列表实时显示 170m 攻击缓冲范围内所有敌人的用户名、HP 和距离；点击列表行会锁定该敌人为攻击对象。
- 锁定目标只要仍在 500m 视野内且存活，就不会被自动切换，即使暂时超出 150m 射程也保持锁定。
- 锁定目标超出 150m 时自动攻击暂停并显示“锁定超出射程”；目标回到射程内后继续按预判攻击。
- 如果锁定目标离开 500m 视野或不再存活，脚本自动解除锁定，并恢复 HP 最低、距离最近的自动目标选择。
- 未手动锁定时，只自动选择 150m 红圈范围内的存活敌人，优先选择 HP 最低者，HP 相同则优先距离更近者。
- 瞄准点按目标当前位置、最近运动轨迹或实体速度字段、相对距离和子弹速度计算短时拦截预判。
- 子弹速度优先读取页面状态里的 projectile/bullet 速度字段，其次使用近期观察到的弹体速度，最后使用游戏默认 `speed_per_tick=500`、`serverTickMs=50` 推算出的约 `10000cm/s`。
- 点击位置复用现有自适应 `worldToScreen` / `viewParams` 换算，不使用固定屏幕坐标。
- 每组攻击使用长按连发形式：一次按住鼠标并覆盖发射 5-8 发，组内按约 `100ms` 的游戏最大射速推进瞄准点，不做单发随机点射。
- 远距离目标会在预测点附近加入垂直/前后覆盖偏移，让一组子弹覆盖目标运动路径周边，而不是只追求单发精确命中。
- 每发按 `0.5` 体力估算，5 秒体力池按 `10` 上限处理；每组可以消耗较多体力，允许偶尔把短时体力打到 0，低于一发体力时才不启动新一组。
- 自动攻击只在真正开火时更新游戏指针和派发鼠标事件，不参与连线/标记渲染循环。

## 受伤离开逻辑

常态金币巡航模式下，如果当前 HP 小于上次记录的 HP：

1. 停止移动。
2. 点击页面“离开”按钮脱战。
3. 停止脚本移动状态。

PC 端脚本处于“停止”模式时，仍会通过 HUD 状态刷新保留 HP 监控。停止态首次看到当前玩家实体时记录 HP 基线；之后只要 HP 低于该基线，就立即点击“离开”脱战。该逻辑只负责保命，不会恢复金币巡航、追杀、交战或任何移动控制。

## 1h 体力限制离开逻辑

脚本会每秒读取游戏左侧官方 `.side` 面板文本。如果左侧栏出现 `1h体力限制`，立即点击“离开”退出游戏。

这个检测不依赖金币巡航是否启动；只要 userscript 已经加载并找到游戏变量，就会生效。它不会读取聊天区，避免被历史聊天消息误触发。

## 金币路线规划

金币巡航不是单纯最近优先，也不是只看单个目标。脚本会先生成安全金币候选池，再比较“单个高价值/近距离金币”和“金币团连续路线”的单位时间收益。

路线评分综合：

- 金币金额
- 连续多个金币之间的路程估计
- 周边金币密度
- 目标点距离富敌的安全程度
- 每段路径中点距离富敌的安全程度
- 是否需要明显掉头
- 当前目标稳定性偏置

金币密度高时，路线会规划更多连续金币点；金币稀疏时，只规划较短路线或单点目标。这样会优先吃一片金币团里的顺路金币，减少吃完一个点后重新掉头找下一个点。

默认约每 1.8 秒重规划一次。若当前路线仍然有效，只有新路线评分明显更好才切换；当路线上的当前金币消失后，会立即允许重新评估剩余路线。

## 右键手动坐标

在游戏画布上点击鼠标右键时，脚本会读取游戏页面当前 `pointerWorld` 坐标，并设置为手动目标。

手动目标行为：

- 如果脚本未启动，会自动启动。
- 安全逻辑仍然优先，包括受伤离开、体力、死亡、170m 敌人规避。
- 安全时会暂停金币规划，优先前往右键坐标。
- 手动开启临时交战会先取消既有右键目标；临时交战模式中重新右键后，右键坐标优先于自动躲避/远离危险敌人，但低血量离开和死亡停止仍然优先。
- 到达目标约 160cm 范围内后，自动清除手动目标，并恢复金币规划。

## 自动追杀

HUD 左下角的“追杀用户名”输入框用于输入用户名片段。点击“追杀”或在输入框内按 Enter 后，脚本会开启自动追杀。

匹配规则：

- 只匹配玩家用户名字符串，不匹配 `user_id`。
- 优先精确匹配，其次前缀匹配，再其次包含匹配。
- 实时实体必须带有 `name`，或能从 `state.userNames` 取到已知用户名，才会参与匹配。
- 没有实时实体时，minimap（小地图）快照只有在该玩家已有已知用户名时，才会作为最后已知位置参与追杀。
- `user_id` 只用于锁定同一个目标和辅助显示，不作为匹配依据。

追赶逻辑：

- 如果目标实体带有 `vx/vy`，按当前服务器 tick（游戏刻）换算为速度。
- 如果实体没有速度字段，则使用最近两次观察到的位置差估算速度。
- 根据距离使用约 350-1300ms 的短预判窗口，追向预判点，而不是只追当前坐标。
- 到达约 260cm 内时停止强行移动，保持观察并继续更新目标。

优先级：

- 受伤离开、死亡停止和体力检查仍然高于追杀。
- 临时交战模式高于追杀；交战开启时追杀会暂停，关闭交战后继续追杀。
- 追杀高于普通 170m/220m 富敌规避和金币巡航，否则目标进入近距离时会被巡航规避逻辑反向拉走。
- 开启追杀会清除已有右键坐标目标；追杀开启后再次右键，会关闭追杀并切换为右键坐标移动。

## 移动策略

脚本保留八方向移动，但会按目标斜率选择：

- x 方向占优：横走
- y 方向占优：竖走
- 两者接近：斜走

这样避免一直 45 度斜线前进。
